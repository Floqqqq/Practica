# Контекст — frontend

React-интерфейс интеллектуальной поисковой системы по внутренней базе знаний университета. Реализация закрывает требования FE-01–FE-09 из задания на учебную практику.

## Возможности

- множественная загрузка PDF и DOCX через Drag-and-Drop или системный выбор файлов;
- клиентская валидация формата, пустого файла и лимита 20 МБ;
- отдельный прогресс каждого файла со стадиями «Загрузка», «Индексация», «Готово», «Ошибка»;
- список документов с названием, датой, размером и статусом;
- полнотекстовый поиск по кнопке и Enter;
- карточки результатов с файлом, страницей, фрагментом и релевантностью;
- подсветка всех слов поискового запроса;
- пагинация по 10 результатов;
- локальная история последних восьми запросов;
- адаптивная верстка от 320 до 1920 px.

## Запуск

Требуется Node.js 20+.

    cd frontend
    npm install
    npm run dev

Приложение откроется на http://localhost:5173. Vite проксирует /api и /health на http://localhost:8080. Другой адрес задаётся в локальном файле .env:

    VITE_PROXY_TARGET=http://localhost:8080
    VITE_API_URL=

## Проверки и production-сборка

    npm run typecheck
    npm run build
    npm run preview

Для контейнерной сборки:

    docker build -t knowledge-search-front ./frontend
    docker run --rm -p 8081:80 knowledge-search-front

В Docker-конфигурации Nginx ожидает сервис бэкенда с DNS-именем backend на порту 8080 и проксирует к нему /api.

## Контракт API

Загрузка использует POST /api/v1/documents/upload, multipart-поле file. Текущий бэкенд принимает один файл за запрос, поэтому множественная загрузка реализована очередью независимых запросов.

Поиск использует GET /api/v1/search с параметрами q, page и limit. Клиент понимает как массив результатов, так и объект с results/items, total, page и total_pages. Поля результата нормализуются из chunk_id, file_name, page/page_number, text и score.

Пока на бэкенде нет эндпоинта списка документов, завершённые загрузки сохраняются в localStorage браузера. Это обеспечивает FE-03 без фиктивных серверных данных; после появления GET-эндпоинта хранилище можно заменить в services/storage.ts.
